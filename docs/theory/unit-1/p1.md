# Introduction to JFC and  Swing

## What is JFC?
Java Foundation Classes (JFC) is a comprehensive suite of graphical user interface (GUI) components and services designed to simplify the development and deployment of desktop, client-side, and internet applications. 

**Java Foundation Classes(JFC)** is a set of GUI components and features provided by Java to help developers build rich, platform-independent graphical user interfaces (GUIs).

JFC is not just Swing — it is a **larger umbrella** that includes:

1. **AWT (Abstract Window Toolkit)** – The original GUI toolkit of Java.  It provides a basic set of controls, windows, and dialog boxes that are used to create user interfaces.
2. **Swing** – Lightweight GUI components built on top of AWT. Swing provides a much larger set of components than AWT, including tables, trees, and tabbed panes.
3. **Java 2D API** – For advanced 2D graphics, text, and images.
4. **Accessibility API** – Helps build GUIs usable by people with disabilities (screen readers, alternate input devices, etc.).
5. **Drag and Drop Support** – Enables data transfer within and between Java applications.
6. **Pluggable Look and Feel (PLAF)** – It allows an application's GUI to look like Windows, macOS, or Linux without changing the application's source code.

So JFC is a collection of GUI components, services, and APIs that provide a consistent and unified way to build GUIs for Java applications.



## What is Swing?

Swing is a part of the Java Foundation Classes (JFC) that provides a rich set of Graphical User Interface (GUI) components for developing desktop applications in Java. It extends the capabilities of AWT by offering more advanced, flexible, and customizable components. Since Swing is written entirely in Java, it is platform-independent, allowing applications to have a consistent look and feel across different operating systems.

### Key Features of Swing
- **Part of JFC:** Swing is a core component of the Java Foundation Classes (JFC).
- **Extends AWT:** It enhances Abstract Window Toolkit (AWT) by providing more powerful and feature-rich GUI components.
- **Lightweight Components:** Swing components are lightweight because they are written entirely in Java and do not rely on the operating system's native GUI components.
- **Platform-Independent:** Swing applications run on any platform that supports Java and maintain a consistent appearance.
- **Rich Set of Components:** It provides a wide variety of GUI components such as `JButton`, `JLabel`, `JTextField`, `JTable`, `JTree`, `JTabbedPane`,`JApplet`, J`Frame`, and many more.
- **Highly Customizable:** Developers can easily customize the appearance and behavior of components using borders, icons, colors, fonts, and layouts.
- **Pluggable Look and Feel (PLAF):** It allows the GUI to look like Windows, macOS, Linux, or other themes without changing the application code.
- **Package:** Most Swing classes are available in the `javax.swing` package.


Swing is a powerful and flexible GUI toolkit in Java that simplifies the development of modern desktop applications. With its lightweight, platform-independent components and extensive collection of GUI controls, Swing is widely used to build attractive, interactive, and user-friendly Java applications.
